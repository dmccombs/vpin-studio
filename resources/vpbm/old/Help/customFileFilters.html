Custom File Filters Panel

Virtual Pinball Backup Manager - managing your backups, one game at a time

  Overview

At times there will be games for which the default file filters do not pick up all associated emulator game content. This may occur because the game uses multiple ROMs (in VPX) or perhaps there are specific library files required for the game (Future Pinball).

The Custom File Filters panel is designed to address this sort of situation by allowing users to add their own additional game specific filters directly targeting these missing files. These filters, once added are stored with the game, both in the backup file and in the Pinup System database.

Custom File Filters are only designed to work within the emulator directory structure, the base emulator directory for the selected game is shown in the Emulator Base Path text field.

As custom file filters are modified, content in the Selected Game Content panel are immediately updated to reflect.

In addition to managing custom filters, the Custom File Filters panel provides a summary of path variables that are available for use. These variables are extracted from table scripts. For variables other than [GAMENAME] a facility is provided for users to override individual variables for a given game. This is provided to handle cases where automatic extraction of variables is not possible, or producing incorrect results.

  Defining a Filter

Custom File Filters are made up of two elements, shown below:

Path
A path relative to Emulator Base Path to perform the filtering in. Wildcards (*) are not permitted in this path.
Filter
A filter to apply to the contents of Path to identify game content. A filter may match to a filename within Path or to a directory name. When a match is made to a directory, the directory and all its content is included as game content. Wildcards (*) may be included in this filter, however path separators (\) are not.

Both Path and Filter values may make use of VPBM path variables, described below to represent parts of the path/filename. The calculated values of these variables is displayed in the bottom of the Custom File Filters Panel.

In the example image the calculated values are 007 Goldeneye (Sega 1996) VPinWorkshop 1.0 MOD VR for [GAMENAME] and gldneye for [GAMECODE]. The [GAMECODE] value is used in the second example filter to look for a directory inside the VPinMAME\artwork directory matching the [GAMECODE] value so would look for files or directories called gldneye.


  Managing Custom File Filters

Three buttons are provided to allow users to Create new, Update or Delete existing custom filters. These buttons will become available when they are valid to use.

Once defined custom file filters will be considered along with the default file filters for the games emulator and influence which files are considered part of the game.

  Making Custom File Filters Global

Should it be determined that a custom filter should be applied to all games for the current emulator, then the filter should be added globally via the VPBM configuration file, as described in Application Settings.
